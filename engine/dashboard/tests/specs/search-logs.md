# Search Logs Page

Logs are generated by navigating the dashboard (each page load triggers API calls).

## search-logs-1: Logs appear after API calls (SMOKE)
1. Go to /index/e2e-products (triggers API calls)
2. Navigate to /logs
3. See the "API Logs" heading
4. See the request count badge (e.g., "5 requests")
5. See log entries in the list
6. See each entry showing: time, status icon, HTTP method, URL, and duration

## search-logs-2: Expand log entry details
1. Go to /index/e2e-products (generate some API calls)
2. Navigate to /logs
3. Click on a log entry row
4. See the expanded detail card appear below the row
5. See "Response" section with formatted JSON
6. See "Curl Command" section with the curl equivalent
7. Click the same row again
8. See the detail card collapse

## search-logs-3: Toggle curl view
1. Navigate to /logs (with existing log entries)
2. See the view toggle with "Endpoint" and "Curl" options
3. See "Endpoint" mode active by default (showing method + URL per row)
4. Click "Curl" to switch view mode
5. See each log entry now displayed as a full curl command
6. See each curl command starting with "curl -X"
7. Click "Endpoint" to switch back
8. See entries return to the method + URL format

## search-logs-4: Clear logs
1. Go to /index/e2e-products (generate API calls)
2. Navigate to /logs
3. See log entries in the list
4. Click "Clear" button
5. See the log list disappear
6. See the empty state: "No API logs"
7. See the badge show "0 requests"

## search-logs-5: Export logs
1. Go to /index/e2e-products (generate API calls)
2. Navigate to /logs
3. See log entries in the list
4. Click "Export" button
5. See a file download triggered

## search-logs-6: Filter log entries
1. Navigate to /logs (with existing log entries)
2. See the filter input with placeholder "Filter by URL, method, or body..."
3. Type "health" in the filter input
4. See only log entries with "health" in their URL displayed
5. Clear the filter
6. See all log entries displayed again
